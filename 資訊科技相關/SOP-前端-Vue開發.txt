下載程式儲存庫Respository(by git)
  
  1. cd 至 <本機Respository儲存路徑> ex. cd C:\Projects\Projects_NodeJS
  2. git clone <Respository 網址> ex. git clone https://github.com/kuanglian1000/hello-world.git

更新程式最新版內容 git pull
  3. git pull 
    註: 指令必須在<本機Respository儲存路徑>下,
        不然會出現異常訊息 

    異常: "fatal: not a git repository (or any of the parent directories): .git"
    正常: Already up to date.
    正常: Fast-forward
          aNewFile.txt | 1 +
          1 file changed, 1 insertion(+)
          create mode 100644 aNewFile.txt

檢查本地端與GIT端的檔案狀態 git status
  4. git status 
    
    [本地端(aNewFile.txt)內容變動, GIT端未更新, 執行 git status 後的訊息]
      ==Commnad==
      git status
      ==Message==
      On branch main
      Your branch is up to date with 'origin/main'.

      Changes not staged for commit: #注意, 表示檔案內容已變更, 告訴你可以 [追蹤變更](git add <file>) 或是 [捨棄變更](git restore <file>)#
        (use "git add <file>..." to update what will be committed)
        (use "git restore <file>..." to discard changes in working directory)
              modified:   aNewFile.txt

      no changes added to commit (use "git add" and/or "git commit -a")

追蹤變更 git add -A
  5. [追蹤變更](git add <file>) #注意: 只要放入{暫存區staged}, git會開始追蹤在工作區{Working Folder}內的每次變更.

      git add aNewFile.txt #只將aNewFile.txt的變更放入{暫存區staged}
      git add -A #所有變更檔案都放入{暫存區staged}

      ==Commnad==
      git add -A 
      
      ==Message(git status)==
      On branch main
      Your branch is up to date with 'origin/main'.

      Changes to be committed: #注意: 放入{暫存區staged}後, 可以 [取消變更追蹤](git restore --staged <file>), 就是git不會理你怎麼改了..#
        (use "git restore --staged <file>..." to unstage)
              modified:   aNewFile.txt

捨棄變更/清除變更 git restore 
  6. [捨棄變更](git restore <file>) #注意: 將{本地端檔案變更}內容清除, 恢復為儲存庫Respository內容, 執行前須十分小心

      ==Commnad==
      git restore aNewFile.txt #捨棄aNewFile.txt的變更, 恢復為儲存庫Respository內容

取消變更追蹤
  7. [取消變更追蹤](git restore --staged <file>) #注意: 移出{暫存區staged}後, git就不會理你要怎麼改了..

      ==Commnad==
      git restore --staged aNewFile.txt #移出{暫存區staged}, 內容不會變動.

版本提交
  8. [版本提交](git commit -m "版本資訊...")

      ==Commnad==
      git commit -m "這個版本改了什麼東西" #{暫存區staged}內所有變動作為一版提交.

      ==Message==
      [main 0504425] add new words..
      1 file changed, 8 insertions(+)

送入儲存庫/送至線上的GitHub
  9.[送入儲存庫](git push origin <branch_name>)

      git push origin main

npm(Node.JS 套件管理程式), 下載Vue也是透過npm #注意: npm install 要搭配 package.json使用, 不然它不知道要下載什麼.

  1.下載 Node.JS # 記得將安裝路徑加入系統變數中, npm才能到處執行
  2.npm install 尋找 package.json 內定義, 下載所需套件項目.

切換至專案路徑, 下載必要套件 ex. C:\Projects\Projects_NodeJS\30days-challange

  1.cd C:\Projects\Projects_NodeJS\30days-challange
  2.檢查專案路徑有沒有 package.json
  3.執行下載作業 npm install
